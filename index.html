Security and trust

Power without visible boundaries does not feel premium. It feels risky.

A serious AI operator needs guardrails built into the experience: scoped permissions, approval for risky actions, run traces, exportability, and quota visibility so users understand both value and limits.

A

Approval-first on sensitive actions

External publishing, deletion, purchases, and other irreversible actions should always pause for review instead of pretending autonomy is the product.

B

Visible daily limits

A free tier should be real and useful, but limits should be transparent. That creates trust now and a clean path to paid plans later.

C

Support is part of the system

Customer support, investigation, and run debugging belong inside the product strategy, especially for agents that act on behalf of users.

Pricing

FAQ

What matters before people trust an agent product.

Is this already the full backend product?

Not yet. This site and workspace are currently high-quality front-end prototypes for the product direction: global positioning, app structure, workflow logic, pricing philosophy, and trust design.

Can it support many software integrations?

That is the intention. The product direction assumes broad OAuth-based connectors plus extensibility through compatible tool protocols and internal integrations.

Can it control a computer?

That is part of the roadmap, but browser execution should come first. Full computer control needs stronger safety, traceability, and approval design before it is credible.

Will it stay free forever?

The better promise is not “everything free forever.” The better promise is a permanent free tier with real value, then paid expansion for power and scale.
